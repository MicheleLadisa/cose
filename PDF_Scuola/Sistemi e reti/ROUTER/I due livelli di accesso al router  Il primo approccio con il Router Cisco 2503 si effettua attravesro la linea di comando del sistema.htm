I due livelli di accesso al router

Il primo approccio con il Router Cisco 2503 si effettua attravesro la linea di comando del sistema operativo Cisco IOS.
Il sistema operativo da la modalità di accesso a diversi livelli:

  • Il livello utente : include servizi riguardanti il controllo del funzionamento del router (con questa modalità di accesso le modifiche alla configurazione non verranno accettate).
  • Il livello privilegiato : include servizi riguardanti la configurazione "avanzata" del router.
La prima volta che si accederà al router si accederà in modalità utente, se osservate bene il metodo di accesso è indicato dal prompt, che in questo caso sarà del tipo:

Router>

Per accedere al livello di configurazione bisogna utilizzare il comandoenable, che richiederà la password (che potrà essere settata successivamente con il comando enable secret). Completata la procedura di Login il prompt è cambiato, sarà infatti det tipo:

Router#

Per disattivare la modalità di configurazione digitare disable, che uscirà così dal livello privilegiato e ritornerà in modalità utente.

N.B. Per effettuare il logout utilizzare il comando exit (da aggiungere immagine del prompt di login).

La funzione di help 
Per controllare la sintassi di un qualsiasi comando si utilizza la potente funzione di help del sistema operativo IOS della Cisco:

ad esempio se dovessimo settare la data del Router possiamo digitare "c?" che restituisce la lista di tutti i comandi che cominciano appunto per c. Individuato adesso il comando da utilizzare (clock) dobbiamo conoscerne la sintassi, per fare ciò utilizzaremo ancora il "?" secondo questo modello:

comando ? 

oppure se il comando ha parametri 

comando [lista parametri] ?

Il router risponderà inviando la lista dei possibili valori del parametro sconosciuto o quello che significano e il formato di inserimento.

N.B. Il carattere "^" indica un errore di sintassi. 

La cronologia dei comandi dell' IOS

In modo completamente autonomo il Router tiene traccia in un buffer, detto "command history", degli ultimi 10 comandi digitati. La capacità del Buffer è dimensionabile dall'utente fino ad un massimo di 256 righe tramite il comando "history size" o "terminal history size" se si opera da un terminale.

Ctrl-P o up arrow key Recalls last (previous) command
Ctrl-N o down arrow key Recalls most recent command
Router> show history Shows command buffer
Router> terminal history size numbers of lines Sets command buffer size
Router> no terminal editing Disables advanved editing features
terminal editing Re-enables advaced editing
<tab> Completes the entry
