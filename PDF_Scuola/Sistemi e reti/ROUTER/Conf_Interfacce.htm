Configuriamo le interfacce

Router#conf ?
memory Configure from NV memory
network Configure from a TFTP network host
overwrite-network Overwrite NV memory from TFTP network host
terminal Configure from the terminal
<cr>
 

Il comando configure indica il luogo dal quale prelevare la configurazione delle interfacce, di solito si indicano i parametri mediante il terminale, useremo quindi il comando configure terminal o abbreviandolo in conf t.

Router#conf t
Enter configuration commands, one per line. End with CNTL/Z.
Router(config)#

Per uscire dalla modalità configurazione e rientrare nella modalità utente si usano i comandi Exit o la pressione dei tasti Ctrl/Z.

Il nome del router, che compare all'interno del prompt, può essere assegnato con il comando HostName, in modalità privilegiata

Router(config)#HostName Tokyo
Tokyo(config)#

Si nota il diverso prompt; ora indichiamo quale delle interfacce desideriamo modificare tra quelle indicate (S0 S1 E0) dal progetto della connessione di rete.

Router(config)#int e0
Router(config-if)#

e forniamo la configurazione minima per questa interfaccia:

Router(config-if)#ip address 192.168.1.90 255.255.255.0
Router(config-if)#no shutdown
Router(config-if)#^Z
Router#
%LINK-3-UPDOWN: Interface Ethernet0, changed state to up
%SYS-5-CONFIG_I: Configured from console by console

L'indirizzo IP è seguito dalla maschera della sottorete a cui appartiene il router, si indica quindi il desiderio di attivare la periferica ed infine si esce dalla configurazione dell'interfaccia; il sistema operativo risponde attivando lo stato dell'interfaccia ethernet0.

Si dovrebbe quindi verificare se la connessione è attiva, colleghiamo un cavo alla presa ethernet e verifichiamo con un ping lo stato della linea.
