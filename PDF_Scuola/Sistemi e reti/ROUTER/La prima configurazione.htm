La prima configurazione:

Quando si accende il router il sistema operativo e i file di configurazione vengono copiati dalla NVRAM alla RAM.
Se l' OS non è presente nella NVRAM il router interroga ogni periferica con comunicazione a pacchetto finoa trovare un router che gli spedisca il sistema operativo via network; la richiesta viene effettuara mediante una interrogazione Brodcast e la trasmissione avviene mediante TFTP.

Adesso il router non ha nessuna configuraizone Ethernet. L'unico modo di accedere al router è tramite console, si collega il router con cavo rollover e, all'altro capo si usa un adattatore da RJ45 a porta COM. A questo punto si apre una connessione con un programma terminale alla seriale RS232 a cui è collegato il router con impostazioni 9600,n,8,1. ALimentiamo il router e in pochi secondi vengono trasmessi i messaggi seguenti.

Esaminiamo alcuni di questi comandi (quelli più usati): enable - show - ping - telnet - traceroute

Ogni comando può essere scritto in forma abbreviata ed è possibile richiedere l'aiuto per il completamento dei comandi con i parametri relativi, scrivenda ad esempio sh in vece di show   otterremo lo stesso risultato, inoltre scrivendo sh ? otterremo come risposta un elenco di possibili parametri o comandi per il completamento dell'istruzione.

enable consente il passaggio al livello di sicurezza superiore, quello dal quale si possono modificare le impostazioni delle interfacce, inizialmente non è impostata nessuna password, eventualmente richiesta si usi come password la parola cisco.

show esamina la situazione di funzionamento, il comando deve essere completato con uno dei parametri seguenti:

Router>sh ?
clock Display the system clock
history Display the session command history
hosts IP domain-name, lookup style, nameservers, and host table
isdn ISDN information
ppp PPP parameters and statistics
sessions Information about Telnet connections
snmp snmp statistics
terminal Display terminal configuration parameters
users Display information about terminal lines
version System hardware and software status
 

ping è usato per verificare la connessione con un host o una interfaccia dotata di indirizzo IP , deve essere completato con l'indirizzo IP del corrispondente.

Router>ping 192.168.0.1

Telnet richiama il client telnet per aprire una connessione con un altro host , deve essere completato con l'indirizzo IP del corrispondente.

Traceroute visualizza gli indirizzi IP di tutti i punti di passaggio del pacchetto attraverso la rete fino alla destinazione desiderata , deve essere completato con l'indirizzo IP del corrispondente.

La cronologia dei comandi dell' IOS

In modo completamente autonomo il Router tiene traccia in un buffer, detto "command history", degli ultimi 10 comandi digitati. La capacità del Buffer è dimensionabile dall'utente fino ad un massimo di 256 righe tramite il comando "history size" o "terminal history size" se si opera da un terminale.

Ctrl-P o up arrow key Recalls last (previous) command
Ctrl-N o down arrow key Recalls most recent command
Router> show history Shows command buffer
Router> terminal history size numbers of lines Sets command buffer size
Router> no terminal editing Disables advanved editing features
terminal editing Re-enables advaced editing
<tab> Completes the entry
